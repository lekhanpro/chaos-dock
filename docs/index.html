Chaos-Dock is a Go + Bubble Tea CLI that finds running containers, injects controlled faults, and validates recovery behavior with OS-level traffic controls.

Explore Repository Run Locally
chaos-dock session 
$ chaos-dock explore
containers: api postgres redis

$ chaos-dock inject latency --target postgres --delay 500ms
status: qdisc netem delay 500ms active on eth0

$ chaos-dock panic
status: reverted latency + restarted 1 container

How It Works

OS-level mutation, app-level validation

  1. Docker SDK inspects target container and resolves host PID from State.Pid.
  2. Injector enters the target namespace with nsenter --target <pid> --net --mount.
  3. Runs tc qdisc replace dev eth0 root netem delay 500ms via exec.CommandContext.
  4. Panic button removes qdisc and triggers restart flow for impacted containers.

Quickstart

Run your first experiment

Sample chaos.yaml

experiments:
  - name: db-latency
    targetContainer: postgres
    enabled: true
    fault:
      type: network-latency
      delay: 500ms
    schedule:
      every: 60s

Run Commands

git clone https://github.com/lekhanpro/chaos-dock.git
cd chaos-dock
go run ./cmd/chaos-dock

Requires Linux host privileges for namespace networking and tc execution.
